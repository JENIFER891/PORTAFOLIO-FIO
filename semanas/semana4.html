Semana 4: Tailwind CSS

Framework de utilidades CSS para crear interfaces modernas de forma rápida y eficiente.

Concepto de clases utilitarias (Atomic CSS)

Definición y características

Tailwind CSS utiliza un enfoque de "Atomic CSS" donde cada clase representa una única propiedad CSS.

.card { background: white; border-radius: 0.75rem; padding: 2rem; box-shadow: 0 4px 6px -1px rgba(0,0,0,0.1); } <div class="bg-white rounded-xl p-8 shadow-md"></div>

Ventajas

  • Reducción de CSS personalizado
  • Consistencia visual en todo el proyecto
  • Facilidad para diseño responsivo
  • Mejor mantenibilidad del código
Diseño responsivo con sm, md, lg, xl

Breakpoints predefinidos

Prefijo Pantalla mínima Uso común
sm: 640px Teléfonos grandes
md: 768px Tablets
lg: 1024px Laptops
xl: 1280px PCs de escritorio

Ejemplo práctico

<div class="text-sm md:text-lg xl:text-xl"> Este texto cambia de tamaño según el dispositivo </div>
Personalización con tailwind.config.js

Configuración básica

Tailwind permite personalizar colores, fuentes, espaciados y más mediante el archivo de configuración.

// tailwind.config.js module.exports = { theme: { extend: { colors: { primary: '#1E40AF', secondary: '#FBBF24', } } } }

Uso de colores personalizados

<button class="bg-primary text-white"> Botón con color personalizado </button>
Espaciado, tipografía y bordes

Clases comunes

Tipo Ejemplos Descripción
Espaciado m-4, p-4, mx-2, py-3 Margins y paddings
Tipografía text-lg, font-bold, text-center Tamaño, peso y alineación
Bordes border, border-2, rounded-lg Grosor, estilo y radio

Ejemplo combinado

<div class="m-4 p-4 border-2 border-blue-500 rounded-lg"> Contenedor con espaciado y bordes </div>
Flexbox y Grid

Flexbox con Tailwind

<div class="flex flex-col md:flex-row justify-center items-center gap-4"> <div>Elemento 1</div> <div>Elemento 2</div> </div>

Grid con Tailwind

<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4"> <div class="col-span-1">Item 1</div> <div class="col-span-1">Item 2</div> <div class="col-span-1 md:col-span-2">Item 3</div> </div>
Dark Mode y variantes de estado

Modo oscuro

<div class="bg-white dark:bg-gray-800"> <p class="text-gray-900 dark:text-white"> Texto que cambia en modo oscuro </p> </div>

Variantes interactivas

<button class="bg-blue-500 hover:bg-blue-700 focus:ring-2 focus:ring-blue-300 active:bg-blue-800"> Botón interactivo </button>
Optimización con PurgeCSS

Configuración básica

// tailwind.config.js module.exports = { content: ['./src/**/*.{html,js}'], // Elimina clases no utilizadas en producción }

Modo JIT (Tailwind v3+)

Tailwind v3+ incluye el modo Just-In-Time que genera solo las clases usadas, eliminando la necesidad de PurgeCSS.

Animaciones y efectos

Transiciones y transformaciones

<button class="transition-all duration-300 ease-in-out transform hover:scale-105 hover:shadow-lg"> Botón con efectos </button>

Sombras

<div class="shadow-sm hover:shadow-md transition-shadow"> Card con sombra interactiva </div>
Componentización con @apply

Creación de componentes

/* CSS */ .btn-primary { @apply py-2 px-4 bg-blue-500 text-white rounded-lg hover:bg-blue-700; } /* HTML */ <button class="btn-primary">Botón</button>
Integración con frameworks

Comparación de integración

Framework Método de instalación Ventaja principal

Vue npm install -D tailwindcss Plantillas con binding flexible
Next.js npx create-next-app --example with-tailwindcss Renderizado del lado del servidor
