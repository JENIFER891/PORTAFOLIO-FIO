Semana 11: JSP, JAKARTA EE Y SPRING FRAMEWORK

Conceptos fundamentales de JSP, Jakarta EE y Spring Framework, incluyendo arquitectura, configuración y ejemplos prácticos.

1. Apache Tomcat

Introducción

Apache Tomcat es un contenedor de servlets desarrollado bajo el proyecto Jakarta en la Apache Software Foundation. Implementa las especificaciones de servlets y JSP de Oracle.

Logo de Apache Tomcat

Estructura de Directorios

  • bin: Ejecutables y scripts
  • conf: Ficheros de configuración
  • webapps: Directorio raíz de aplicaciones web
  • logs: Archivos de registro
  • work: Información temporal

Configuración (server.xml)

<Server> <Service> <Connector port="8080" protocol="HTTP/1.1" connectionTimeout="20000" redirectPort="8443" /> <Engine name="Catalina" defaultHost="localhost"> <Host name="localhost" appBase="webapps"> <Context path="" docBase="miAplicacion" /> </Host> </Engine> </Service> </Server>
2. JSP (Jakarta Server Pages)

Conceptos Básicos

JSP es un lenguaje del lado del servidor que permite crear páginas web dinámicas usando Java. Combina HTML con código Java mediante etiquetas especiales.

Sintaxis Principal

Scriptlets

<% // Código Java String nombre = request.getParameter("nombre"); out.println("Hola " + nombre); %>

Expresiones

<p>La fecha actual es: <%= new java.util.Date() %></p>

Declaraciones

<%! private int contador = 0; public void incrementar() { contador++; } %>

Objetos Implícitos

Objeto Descripción
request HttpServletRequest asociado a la solicitud
response HttpServletResponse para la respuesta
session HttpSession de la sesión actual
application ServletContext del contexto de la aplicación
3. Spring Framework

Conceptos Clave

Inversión de Control (IoC)

Spring invierte el control del flujo de la aplicación, gestionando automáticamente los objetos y sus dependencias.

Inyección de Dependencias (DI)

Spring permite inyectar dependencias a través de constructores, setters o directamente en campos.

// Inyección por constructor @Service public class MiServicio { private final MiRepositorio repositorio; @Autowired public MiServicio(MiRepositorio repositorio) { this.repositorio = repositorio; } }

Spring Boot

Spring Boot simplifica la configuración de aplicaciones Spring con convenciones inteligentes y auto-configuración.

Arquitectura típica de una aplicación Spring Boot

Características principales:

  • Servidor embebido (Tomcat, Jetty)
  • Configuración automática
  • Métricas y health checks
  • Spring Initializr (start.spring.io)
4. Ejercicios Prácticos

API REST con Spring

@RestController @RequestMapping("/api/estudiantes") public class EstudianteController { @Autowired private EstudianteRepository repo; @GetMapping public List<Estudiante> listarTodos() { return repo.findAll(); } @PostMapping public Estudiante crear(@RequestBody Estudiante estudiante) { return repo.save(estudiante); } }
